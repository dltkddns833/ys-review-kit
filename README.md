## 영수님 협업 가이드 (상운 개인 메모)

- **리뷰 스타일**: 코드·문서를 한 줄씩 검증하는 깐깐한 리뷰어. 머지 후에도 "왜 이렇게 했냐"고 다시 짚는 경우가 잦음. PR/문서 준비 시 사소한 모순·오타·기준 불일치까지 사전에 잡아둘 것.
- **스킬 작성 기준**: `better-wealth-pb` 저장소의 `skills/mydata` 폴더를 반복적으로 "표준 템플릿"으로 인용함. 이 저장소의 스킬을 작성·리뷰할 때는 mydata의 구조(README 섹션 구성, SKILL.md description 작성 방식, 참조 문서 표 등)와 대조해 일관성을 확인할 것.
- **사용자 요청 패턴**: 영수님 리뷰가 예상되는 작업물을 가져왔을 때는 단순 의견이 아니라 "영수님이 짚을 만한 포인트"를 사전에 정리해서 줄 것.

## 유지보수

- **새 사례 등록**: `references/YYYY-MM-DD-<slug>.md` 파일 생성 후 `SKILL.md`의 `## 참조 문서 안내` 테이블과 이 README의 `## 참조 파일 구조` 블록에 동시 추가. 셋 중 하나라도 누락되면 영수님이 바로 짚음.
- **증거 등급 유지**: 각 사례 파일은 맨 앞에 증거 등급(직접/간접)을 명시한다. 간접 증거만 있을 때는 "영수님은 이렇게 말한다"가 아니라 "영수님 기준이라면 이 부분을 짚을 가능성이 높다"로만 서술.
- **글로벌 사본 동기화**: 이 README와 `~/.claude/CLAUDE.md`는 서로의 사본(어느 쪽이 canonical인지 명시 없음). 한쪽을 고치면 다른 쪽도 맞출 것.
- **파일명·섹션명 변경 시**: `SKILL.md`·`README.md`·`references/` 내부의 상호 참조를 모두 함께 수정.
